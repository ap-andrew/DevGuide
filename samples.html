Apigee API Platform samples are designed to help developers learn how to use the Apigee API Platform. The package contains sample API proxies, policies, code, and tools that illustrate the capabilities of the Apigee API Platform.

Where to get samples

If you have git installed, run:

$ git clone https://github.com/apigee/api-platform-samples

If you do not have git installed, you can download the samples from the API Platform Samples Github Repo.

How to use samples

All of these samples require you to have an account on http://enterprise.apigee.com.

You can sign up for a trial account at http://enterprise.apigee.com/signup.

To use the included deployment tool, you also require Python to be installed on your machine.

Samples and tools

/simpleProxy

A basic API proxy that is used in the API Platform Developer Guide. The Developer Guide shows you how to build API facades that run on the API platform, with instructions on API Keys, OAuth 1 and 2, developer and app provisioning, custom scripting tutorials, environment configuration, and more.

/tools

Contains a deployment tool, deploy.py, that imports an API proxy to an organization on the API Platform and then deploys it to the environment specified.

/sample-proxies

Fully configured API proxies that can be immediately deployed and invoked on the API platform, with examples of:

  • Conditional policy enforcement and routing
  • API key validation
  • Server-side OAuth 2.0 client credentials and authorization code grant types
  • Outbound requests using Auth 2.0 access tokens
  • Caching
  • Composite services, built using policy composition and Javascript, that mashup the Google location and elevation APIs and expose their own API for sharing and reuse.

/schemas

XML schema files that you can use as a reference when configuring API facades, policies, API products, and developer and app profiles.

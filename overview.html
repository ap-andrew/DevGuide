What's here?

The API Platform Developer Guide is designed to get you up and and running with local development of API proxies that deploy and scale on the Apigee API Platform.

The focus of development on the API Platform is the API proxy. An API proxy is an application that runs on the Apigee API Platform and which serves as a proxy for one or more backend services or APIs. Every API proxy exposes a new API that can be consumed by apps.

For an overview of how the API Platform can help you as a developer, Watch the video.

Using API proxies, you can proxy any publicly available API, which makes the API Platform a great solution for developers who need to customize the behavior of APIs--whether the developer controls the API or not. Once you proxy an API with an API proxy, all of the enterprise capabilities of Apigee are at your disposal. You can:

  • Leverage a policy library to implement common behavior without writing any code
  • Innovate on the API Platform by writing and deploying custom code in the Cloud
  • Combine policies and custom code to create sophisticated yet easy-to-manage and secure applications
  • Provision and manage API keys and OAuth authorization
  • Monitor the operational performance of APIs
  • Track and analyze consumption patterns by developers and apps
  • Expose the capabilities that you implement as APIs, complete with an API Console, enabling re-use across projects or sharing with other developers

Background

This Developer Guide provides instruction on local development of API proxies. You don't need to be an expert in any of these topics, but, when building API proxies, it's useful to have some familiarity with the following:

  • Representational State Transfer (REST)
  • JavaScript Object Notation
  • Extensible Markup Language
  • XML Schema
  • Web Application Description Language (WADL)

Optionally:

  • JavaScript and Python scripting languages
  • Java programming language

You can also create API proxies in the API Platform Management UI. For instructions, refer to the the API Platform User Guide.

What you'll learn

In this Developer Guide you will learn:

  • How to configure a basic API proxy
  • How to upload, deploy, and test API proxies on the API Platform
  • How to configure policies and attach them to an API proxy
  • How to write scripts and code that execute as components of an API proxy
  • How to implement common use cases required by API developers
  • How to expose and reuse your policy compositions and code as APIs using WADL and API Consoles
  • How to configure environment settings for your organization on the API Platform
